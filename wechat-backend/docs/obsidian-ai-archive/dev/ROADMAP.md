# 开发路线图

## 总原则

- 不丢主体文字
- 规则必须可解释、可测试、可回归
- 优先保留保真层，再做派生层

## Priority 1

### 1. 正文清洗规则系统化

目标：

- 把现有“代码里散落的安全规则”整理成有层次的规则体系
- 把“不会误删主体文字”做成明确约束，而不是口头原则
- 建立固定样例，后续每次改规则都能回归验证

交付物：

- [CONTENT-CLEANING-DESIGN.md](./CONTENT-CLEANING-DESIGN.md)
- 样例目录规划
- 规则版本化设计

### 2. detached `repair-all` 闭环

目标：

- 后台启动
- 状态查看
- 完成态摘要
- 需要时联动镜像同步

## Priority 2

### 3. 标签化

目标：

- 自动给文章增加主题、来源、时间标签

建议：

- 先做规则标签
- 再引入 LLM 辅助标签
- 结果写回 frontmatter，不改正文

### 4. `AI-clean` 派生层

目标：

- 在保留当前 Markdown 保真层的前提下，再生成一层更适合检索和 embedding 的派生文本

约束：

- 不覆盖原始主文档
- 不破坏 raw sidecar

## Priority 3

### 5. embeddings / 向量索引

目标：

- 基于 `WeWe-RSS-AI` 构建语义检索

### 6. RAG 服务

目标：

- 在本地主库与向量索引之上提供稳定召回与问答

## 建议顺序

1. 固化清洗样例与规则设计
2. 完成 `repair-all` 闭环
3. 做标签化
4. 做 `AI-clean` 派生层
5. 做 embeddings
6. 做 RAG
